# Analysis Report Templates

This directory contains the standard formats for analysis reports generated by the Quantitative Trading skill.

## Templates

1.  [**Technical Analysis**](./technical_analysis.md)
    *   **Use when:** User asks for technical indicators, trend analysis, or a deep dive into a specific stock/ETF without explicit portfolio rebalancing.
             
2.  [**Strategy Signal Analysis**](./strategy_signal_analysis.md)
    *   **Use when:** User asks for trading signals, strategy backtest results, or "what do the strategies say?" about a ticker.

4.  [**Portfolio Optimization**](./portfolio_optimization.md)
    *   **Use when:** User requests a comprehensive assessment of their entire portfolio, including correlation analysis, risk metrics (Sharpe, Volatility), and Markowitz efficient frontier optimization.

5.  [**Risk Assessment**](./risk_assessment.md)
    *   **Use when:** User specifically asks for risk management evaluation, stress testing, VaR (Value at Risk), or risk contribution analysis.


## Usage

When generating a report:
1.  Identify the user's intent (single stock technicals vs. portfolio management).
2.  Select the corresponding template from this directory.
    *   **Custom Reports**: If none of the predefined templates fit the task, feel free to write a custom, well-structured `analysis_report.md` based on actual findings.
3.  **Output the report to `workspace/YYYY-MM-DD/HHMMSS/analysis_report.md`** (in the PROJECT ROOT, follow date-time directory convention).
4.  Read the template content to ensure correct structure.
5.  Fill in the placeholders with calculated data.
